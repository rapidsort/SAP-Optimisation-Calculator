1. Installation

  1. Download this repository as a ZIP and extract it.
  2. Open Google Chrome and navigate to chrome://extensions/.
  3. Enable Developer mode using the toggle in the top-right corner.
  4. Click Load unpacked and select the extracted folder.

2. Pre-Test Analysis

Determine the Minimum Detectable Effect (MDE) or duration before starting a test to ensure your experiment is powered correctly.

  • Find MDE: Calculate the lift you can realistically detect based on your current traffic and a planned test duration.
  • Find Duration: Calculate exactly how many days the test needs to run to detect a specific Desired Lift (%).
  • Traffic Split (%): Specify how much traffic goes to the Challenger (Variant B).
    Example: Entering '10' means a 90/10 split (90% Control, 10% Variant).
    Tip: A 50/50 split is the most "efficient" and will give you results fastest. Custom splits (like 90/10) are great for reducing risk but will require significantly more time/traffic.

3. Post-Test Analysis

Analyze your results to determine if your changes had a statistically significant impact.

  1. Enter total Users for both Control (A) and Challenger (B).
  2. Add your Metrics (the tool supports multiple KPIs) and enter the conversion counts.
  3. Select your Confidence level (95% is standard).
  4. Choose a Test Type:
    • One-Sided: Use if you only care if the Challenger is better than Control. Faster to reach significance.
    • Two-Sided: Use for strict scientific testing to see if there is any difference (better or worse).
  5. Click Calculate Significance for a verdict and recommended action.

Tip: The tool automatically monitors Secondary Metrics as guardrails. If a secondary metric shows a significant negative impact, the tool will issue a "Caution" even if your primary metric is positive.
